# 部署文档与脚本生成 Skill（AI友好版）

> 本文档处于软件生命周期的"测试用例→**部署**"阶段，是全链路的最后一个环节。
> 部署 Skill 负责生成自动化部署脚本和部署文档，面向客户私有化网络环境。
> 本 Skill 为**可选执行**，用户可在编排器中选择是否需要。

---

## 定位与边界

### 上下游关系

| 方向 | Skill | 数据流 |
|------|-------|--------|
| 上游：架构设计 | lyspec-arch | 部署架构图、服务器规划、网络拓扑、中间件清单 |
| 上游：详细设计 | lyspec-detail | 技术选型、数据库类型、依赖服务、构建工具 |
| 上游：代码 | ⑤代码编写 | 项目结构、pom.xml/package.json、端口配置、application.yml |
| 上游：技术选型 | lytech-stack | JDK版本、Node版本、中间件版本规范 |
| 下游 | 无 | 本 Skill 是全链路终点 |

### 职责边界

| 做 | 不做 |
|----|------|
| 生成部署脚本（Shell） | 执行实际部署操作 |
| 生成部署文档（Markdown/Word） | 管理CI/CD流水线 |
| 生成Docker配置（可选） | K8s编排配置 |
| 生成Nginx配置 | 监控系统配置（仅在文档中建议） |
| 生成环境预检脚本 | 云平台特定配置 |

## 全局规则

### 部署模式

| 模式 | 产物差异 | 适用场景 |
|------|---------|---------|
| **裸机部署**（默认） | Shell脚本 + systemd服务文件 | 传统私有化环境，无Docker |
| **Docker部署** | Dockerfile + docker-compose.yml + Shell脚本 | 支持Docker的私有化环境 |

> 默认为裸机部署，符合大部分客户私有化环境现状。

### 脚本编写规范

| 规范 | 说明 |
|------|------|
| Shell兼容性 | 使用 `#!/bin/bash`，兼容 CentOS 7+ / Ubuntu 18.04+ |
| 变量集中管理 | 所有可配置项在脚本顶部以变量形式声明 |
| 错误处理 | 每步操作后检查返回值，失败时输出错误信息并退出 |
| 日志输出 | 关键操作前后输出时间戳 + 操作说明 |
| 幂等性 | 脚本可重复执行，不会因重复执行而出错 |
| 权限检查 | 脚本开头检查是否为 root 用户或有 sudo 权限 |

### 信息提取策略

| 级别 | 条件 | 处理方式 |
|------|------|----------|
| ① 直接提取 | 架构设计/详细设计中有明确描述 | 直接引用 |
| ② AI推导 | 可从项目代码/配置推导 | AI推导并标注 *AI推导*，用斜体标识 |
| ③ 待补充 | 完全无依据 | 标注"待补充"，在用户交互环节集中询问 |

### 文档产物命名规则

| 产物类型 | 命名格式 | 示例 |
|---------|---------|------|
| 部署文档 | `output/{projectName}_部署文档_{YYYY-MM-DD}.md` | `output/项目A_部署文档_2026-03-11.md` |
| 部署脚本目录 | `output/deploy/` | — |
| 环境检查脚本 | `output/deploy/check-env.sh` | — |
| 后端部署脚本 | `output/deploy/deploy-backend.sh` | — |
| 前端部署脚本 | `output/deploy/deploy-frontend.sh` | — |
| 数据库初始化脚本 | `output/deploy/init-db.sh` | — |
| Nginx配置 | `output/deploy/nginx.conf` | — |
| Dockerfile | `output/deploy/Dockerfile` | — |
| docker-compose | `output/deploy/docker-compose.yml` | — |
| systemd服务文件 | `output/deploy/{serviceName}.service` | `output/deploy/project-a.service` |

## AI执行流程

### 阶段1：环境探测

**1.1 读取上游产物**

按优先级读取以下文件，提取部署相关信息：

| 来源文件 | 提取内容 |
|---------|---------|
| 架构设计文档 | 部署架构图、服务器规划表、网络拓扑、中间件清单、端口规划 |
| 后端详细设计 | 技术选型（Spring Boot版本、JDK版本）、数据库类型、依赖中间件 |
| 前端详细设计 | 构建工具（Vite/Webpack）、静态资源部署方式 |
| pom.xml / package.json | 项目坐标、依赖版本、构建命令 |
| application.yml | 端口号、数据库连接、Redis连接、第三方服务地址 |
| lytech-stack | 技术栈版本规范 |

> 若某些上游产物不存在（如跳过了架构设计），从代码和配置文件推导，标注为 *AI推导*。

**1.2 询问部署参数**

```
## 部署参数确认

请确认以下部署信息：

### 1. 部署模式
- A. 裸机部署（默认）— 直接在服务器上安装运行
- B. Docker部署 — 使用容器化部署

### 2. 目标操作系统
- A. CentOS 7/8（默认）
- B. Ubuntu 18.04/20.04/22.04
- C. 其他（请说明）

### 3. 服务器规划
（以下为从架构设计中提取/推导的方案，请确认或调整）

| 服务器 | 用途 | 最低配置建议 |
|--------|------|-------------|
| 服务器1 | 应用服务（后端+前端） | 4C 8G |
| 服务器2 | 数据库（MySQL+Redis） | 4C 16G |

> 单机部署方案：若资源有限，可合并到一台服务器（建议 8C 16G）

### 4. 域名/IP
- 应用访问地址：{待补充}
- 后端API地址：{待补充}

请确认或调整以上信息。
```

> **简化规则**：若项目规模小（功能页面 ≤5，接口数 ≤20），直接推荐单机部署方案，不展开多服务器规划。

**1.3 汇总部署蓝图**

确认后输出部署蓝图摘要：

```
## 部署蓝图

| 项目 | 值 |
|------|-----|
| 部署模式 | 裸机部署 |
| 操作系统 | CentOS 7 |
| 服务器数量 | 1台（单机部署） |
| JDK版本 | 17 |
| Node版本 | 18.x |
| 数据库 | MySQL 8.0 |
| 缓存 | Redis 7.0 |
| Web服务器 | Nginx 1.24 |
| 后端端口 | 8080 |
| 前端端口 | 80（Nginx） |

确认后开始生成部署脚本。
```

### 阶段2：脚本生成

根据部署模式分支执行。

#### 裸机部署模式

**2.1 环境预检脚本 `check-env.sh`**

生成内容：
- 操作系统版本检查
- CPU/内存/磁盘空间检查
- 必要端口占用检查
- 已安装软件版本检查（JDK、Node、MySQL、Redis、Nginx）
- 网络连通性检查（数据库连接、内网DNS）
- 检查结果汇总（PASS/FAIL/WARN）

```bash
# 脚本结构示例
#!/bin/bash
# ============================================
# 环境预检脚本 - {projectName}
# 生成日期：{YYYY-MM-DD}
# ============================================

# ---------- 可配置变量 ----------
REQUIRED_JDK_VERSION="17"
REQUIRED_DISK_GB=20
REQUIRED_MEM_MB=8192
APP_PORT=8080
DB_PORT=3306
REDIS_PORT=6379

# ---------- 检查函数 ----------
check_os() { ... }
check_resources() { ... }
check_ports() { ... }
check_software() { ... }

# ---------- 主流程 ----------
main() {
    echo "====== 环境预检开始 ======"
    check_os
    check_resources
    check_ports
    check_software
    echo "====== 环境预检完成 ======"
    print_summary
}
main "$@"
```

**2.2 后端部署脚本 `deploy-backend.sh`**

生成内容：
- 变量区：JAR路径、JVM参数、日志路径、PID文件
- 构建命令（`mvn clean package -DskipTests`）
- 启动函数（nohup + JVM参数）
- 停止函数（优雅停机 → 超时强杀）
- 重启函数
- 状态检查函数（端口检查 + 健康检查接口）
- 日志查看快捷命令

**2.3 前端部署脚本 `deploy-frontend.sh`**

生成内容：
- 变量区：项目路径、Nginx配置路径、静态资源目标路径
- 构建命令（`npm run build`）
- 静态资源复制到Nginx目录
- Nginx配置更新
- Nginx reload

**2.4 数据库初始化脚本 `init-db.sh`**

生成内容：
- 创建数据库和用户
- 执行DDL（从详细设计产物中的DDL提取）
- 导入初始数据（如有）
- 校验表结构

**2.5 Nginx配置 `nginx.conf`**

生成内容：
- 前端静态资源服务
- 后端API反向代理（`/api/` → `localhost:8080`）
- gzip压缩配置
- 前端路由history模式支持（`try_files`）
- 上传文件大小限制
- 日志配置

**2.6 systemd服务文件 `{serviceName}.service`**

生成内容：
- 服务描述
- 启动命令（ExecStart）
- 停止命令（ExecStop）
- 重启策略（Restart=on-failure）
- 日志配置
- 用户/组配置

#### Docker部署模式

**2.7 Dockerfile**

生成内容：
- 多阶段构建（build + runtime）
- 后端：基于OpenJDK镜像，COPY jar + 启动命令
- 前端：基于Node镜像构建 + Nginx镜像运行
- 健康检查指令

**2.8 docker-compose.yml**

生成内容：
- 后端服务
- 前端服务（Nginx）
- MySQL服务（含初始化脚本挂载）
- Redis服务
- 网络配置
- 数据卷配置
- 环境变量配置
- 健康检查

**2.9 Docker辅助脚本**

- `deploy-docker.sh`：一键构建 + 启动（`docker-compose up -d --build`）
- 包含镜像清理、日志查看、状态检查等辅助命令

### 阶段3：文档生成

生成完整部署文档，共6章。

#### 第1章 部署概述

| 内容 | 说明 |
|------|------|
| 系统简介 | 项目名称、功能概述（从PRD提取） |
| 部署架构图 | Mermaid图，展示服务器/服务/网络关系 |
| 服务器清单 | 表格，含IP（待填写）、用途、配置要求 |
| 端口规划表 | 表格，含端口号、服务名称、协议 |
| 部署文件清单 | 表格，列出所有脚本及用途 |

#### 第2章 环境准备

| 内容 | 说明 |
|------|------|
| 操作系统要求 | 版本、内核要求 |
| JDK安装 | 下载地址（内网包）、安装步骤、环境变量配置 |
| Node.js安装 | 下载地址、安装步骤（前端构建用） |
| MySQL安装 | 安装步骤、初始密码设置、字符集配置 |
| Redis安装 | 安装步骤、密码配置、持久化配置 |
| Nginx安装 | 安装步骤、目录说明 |
| 环境预检 | 执行 `check-env.sh` 的说明 |

> **私有化适配**：所有软件安装步骤提供**离线安装**和**在线安装**两种方式。离线安装使用rpm/deb包，在线安装使用yum/apt。

#### 第3章 后端部署

| 内容 | 说明 |
|------|------|
| 构建步骤 | Maven构建命令、产出JAR路径 |
| 配置文件说明 | application.yml中需要修改的配置项（数据库地址、Redis地址等），表格形式 |
| 部署步骤 | 逐步操作指南 |
| 启停命令 | 启动/停止/重启/查看状态 |
| 日志路径 | 日志文件位置、日志级别说明 |
| 健康检查 | 健康检查接口URL、预期响应 |

#### 第4章 前端部署

| 内容 | 说明 |
|------|------|
| 构建步骤 | npm install + npm run build |
| 环境变量配置 | .env.production中需要修改的配置项（API地址等） |
| Nginx配置 | nginx.conf说明、关键配置项 |
| 部署步骤 | 逐步操作指南 |
| 访问验证 | 浏览器访问URL、预期页面 |

#### 第5章 数据库初始化

| 内容 | 说明 |
|------|------|
| 数据库创建 | 数据库名、字符集、用户权限 |
| 表结构初始化 | DDL执行步骤 |
| 初始数据导入 | 初始数据脚本（如有） |
| 验证步骤 | 表数量检查、关键表数据检查 |
| 备份策略建议 | 定时备份命令、备份保留策略 |

#### 第6章 验证与运维

| 内容 | 说明 |
|------|------|
| 部署验证清单 | Checklist，逐项检查（后端启动、前端访问、数据库连接等） |
| 常见问题排查 | FAQ表格，问题→原因→解决方案 |
| 日志查看 | 各服务日志路径、常用日志查看命令 |
| 启停维护 | 日常启停操作手册 |
| 监控建议 | 建议监控的指标（CPU、内存、磁盘、JVM、接口响应时间） |

## 增量更新模式（B路径适用）

> 当通过 lyspec-change-apply 调用时，以增量更新模式运行。

### 触发条件

以下变更类型**可能**影响部署：
- 新增外部集成（新增中间件依赖）
- 数据模型变更（DDL变更需更新初始化脚本）
- 非功能需求变化（性能/安全要求变化影响部署配置）
- 技术栈变更（JDK/Node版本变化）

### 增量更新行为

| 变更内容 | 更新范围 |
|---------|---------|
| 新增中间件（如MQ） | 环境预检脚本追加检查项 + 部署文档追加安装章节 + docker-compose追加服务 |
| DDL变更 | 更新 init-db.sh 中的DDL语句 |
| 端口变更 | 更新相关脚本变量 + Nginx配置 + 部署文档端口表 |
| 配置项变更 | 更新部署文档中的配置说明表 |
| JVM/Node参数调整 | 更新部署脚本中的启动参数 |

> 不影响部署的变更（如纯业务逻辑修改）不触发本skill。
