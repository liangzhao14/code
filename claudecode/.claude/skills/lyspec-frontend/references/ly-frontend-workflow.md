# 前端开发详细工作流

## 1. 需求分析阶段

### 1.1 需求文档解析

从需求文档中提取以下信息：

- **功能模块列表**：识别所有需要实现的功能模块
- **页面清单**：列出所有需要开发的页面
- **用户角色与权限**：不同角色的页面访问和操作权限
- **业务流程**：核心业务流程和状态流转
- **非功能需求**：性能、兼容性、国际化等要求

### 1.2 UI 设计解析

从 UI 设计稿中提取：

- **页面布局**：整体布局结构（侧边栏、顶栏、内容区等）
- **组件拆分**：识别可复用组件和页面级组件
- **交互细节**：表单校验、加载状态、空状态、错误状态
- **响应式设计**：不同屏幕尺寸的适配方案
- **设计 Token**：颜色、字体、间距等设计变量

### 1.3 API 接口分析

从 API 文档中提取：

- **接口清单**：按模块分组的接口列表
- **请求/响应类型**：生成 TypeScript 类型定义的依据
- **认证方式**：Token 管理、刷新机制
- **错误码**：统一错误处理策略
- **分页/筛选约定**：列表接口的通用参数模式

## 2. 代码生成阶段

### 2.1 类型定义

```
src/types/
├── api.d.ts          # 通用 API 类型（分页、响应包装等）
├── {module}.d.ts     # 按模块划分的业务类型
└── index.ts          # 统一导出
```

生成原则：
- 从 API 响应结构推导类型
- 区分请求参数类型（XxxParams）和响应数据类型（XxxData）
- 列表接口统一使用分页包装类型
- 枚举值使用 `const enum` 或字面量联合类型

### 2.2 API 请求层

```
src/api/
├── request.ts        # Axios 实例与拦截器（通常已有，不重复生成）
├── {module}.ts       # 按模块划分的 API 函数
└── index.ts          # 统一导出
```

生成原则：
- 使用项目已有的 Axios 封装实例
- 每个函数有明确的参数和返回值类型
- RESTful 风格命名：`getXxxList`, `getXxxDetail`, `createXxx`, `updateXxx`, `deleteXxx`

### 2.3 状态管理

```
src/stores/
├── modules/
│   └── {module}.ts   # 按模块划分的 Store
└── index.ts          # 统一导出
```

生成原则：
- 使用 Pinia `defineStore` + Setup Store 风格
- Store 只管理需要跨组件共享的状态
- 页面局部状态留在组件内部

### 2.4 组件

```
src/components/        # 通用组件
├── {ComponentName}/
│   ├── index.vue
│   └── types.ts      # 组件 Props/Emits 类型（复杂时抽离）

src/views/{module}/    # 页面组件
├── index.vue          # 页面入口
├── components/        # 页面私有组件
└── composables/       # 页面私有组合式函数
```

生成原则：
- 单文件组件使用 `<script setup lang="ts">`
- 复杂逻辑抽取为 `useXxx` 组合式函数
- 表格、表单等重复模式优先使用组件库能力
- 保持组件职责单一，避免超大组件

### 2.5 路由

```
src/router/
├── index.ts           # 路由实例
└── modules/
    └── {module}.ts    # 按模块划分的路由配置
```

生成原则：
- 路由懒加载：`() => import('@/views/xxx/index.vue')`
- 路由元信息包含：标题、权限标识、面包屑等
- 嵌套路由反映页面层级关系

## 3. 质量检查阶段

### 3.1 代码检查

- 运行 ReadLints 检查每个生成的文件
- 修复所有 lint 错误
- 确认 TypeScript 类型无误

### 3.2 依赖检查

- 确认所有 import 路径正确
- 确认组件注册/引用关系正确
- 确认 Store 和 API 的调用链完整

### 3.3 交付清单

向用户提供：
- 生成文件列表及说明
- 需要手动配置的内容（环境变量、代理配置等）
- 后续联调注意事项
